## Grok chat (Aetheria)

- **API key name**: Use `XAI_API_KEY` in `.env` to match [xAI](https://docs.x.ai/developers/quickstart) and Vercel AI SDK examples. The code also accepts `GROK_API_KEY` as a fallback for existing setups.

- **Responses API + history**: The UI persists full transcripts under `.data/conversations.json` for durable history and reloads. For each new *user* turn after the first reply, the server passes xAI’s `previous_response_id` (stored per conversation as `lastResponseId`) so Grok can continue the server-side chain without resending the entire transcript. Regeneration requests omit `previous_response_id` so the model sees a fresh full `input` for that interaction.

- **Vercel AI SDK**: We use `streamText` with `@ai-sdk/xai` `xai.responses(model)` so streaming, multi-step tool execution, and xAI-specific options stay aligned with current provider support.
- **Default Grok model** (`lib/xai.ts`): `grok-4-1-fast-non-reasoning` when `XAI_MODEL` is unset — fastest/cheapest tier for streamed chat and NationForge. Set `XAI_MODEL=grok-4-1-fast-reasoning` or `grok-4.20-0309-reasoning` if you want heavier reasoning at higher cost/latency.

## NationForge (game)

- **Authoritative state**: Sessions live in `.data/nationforge-sessions.json` with nations, crisis, turn log, Grok thread ids, and secrets. The UI polls `GET /api/nationforge/sessions/[id]` for LAN-style sync (MVP).
- **Stats and math**: The GM must change the six Key Stats and reserve only through the `apply_stat_deltas` tool (per-nation, L1 movement cap per invocation). Narrative stays in streamed text; the board reads numbers from the saved session after each turn.
- **Join model**: `roomCode` for discovery; `POST /api/nationforge/nations` with `{ roomCode, displayName? }` creates a nation row and returns a per-seat `token` for the **nation forge** (not an “official” table seat until `forgeComplete`). Empty `displayName` yields a provisional `Unnamed seat {id}` until the forge naming step, where an AI-suggested name (overridable) is saved via `submitNationName`. Optional `?token=` on the session URL scopes secrets to that seat. New sessions start in `lobby` with zero nations; the first crisis appears only after `gameStarted` flips when every nation in the room has completed the forge once.
- **Official seats vs builders**: `filterSessionForClient` only includes other players’ nations in `nations` when `forgeComplete`; each viewer always sees their own row even while still forging. `nationRoster` lists every slot (id, name, forge flag) so hosts can still copy tokens for in-progress builders. Session list API only joins forged names in `nationNames` and reports `nationsInForge` separately. Bilateral diplomacy requires **both** nations forged.
- **100-point nation builder**: Options and costs live in `lib/nationforge/nation-forge-catalog.ts`; spend and caps are enforced server-side in `POST .../sessions/[id]/forge`. Synergies and Key Stats mapping live in `nation-forge-resolve.ts` so the GM prompt and UI stay aligned with authoritative numbers. Mid-session joiners stay in `forgeComplete: false` until they finish the same stepped flow; `validatePlayerTurn` blocks their turns until then.
- **Review chronicle**: On the confirm step, `loadReviewNarrative` persists `reviewNarrativeMarkdown` (Grok via `generate-forge-review-narrative.ts`, Markdown) for a player-facing briefing; stats remain authoritative. Heuristic Markdown fills in when no API key. Leaving review (`back`) clears the cached narrative.
- **Post-forge onboarding**: When the table first opens (`gameStarted` + crisis, no GM prose yet), the **active seat** auto-sends one `orientationRequest` turn (see `opening-brief-narrative.ts` + `validatePlayerTurn`) so Grok introduces the forged nation before players see the full storyline / crisis form. The crisis banner and turn composer stay hidden until the first GM reply streams in.
- **GM “worldbuilding” UX**: Board shows a full-width banner when `phase === gm_running`, local `busy`, or stream text exists; polls at **650ms** during `gm_running` vs **2.5s** otherwise. Benign duplicate-turn API errors (queue / still writing) trigger a refresh without scary red errors; `openingBriefInFlight` + post-success `openingBeatAutoKeySent` reduce double POSTs.
- **Budget dead-ends**: Every exclusive pillar includes an explicit **0 pt “underfunded / improvised”** choice so players can bank reserve or recover when earlier spends leave too little for priced tiers; those picks apply negative stat tilts in the resolver. The wizard sorts affordable options first and disables picks that exceed remaining points.
- **Back behavior**: `back` clears all selections for steps **after** the step you return to, so totals match the screen; per-step budget uses `computeSpendExcludingStep` so re-picking a pillar does not double-count. The wizard highlights the saved choice on each pillar.
- **Rate limits**: `POST /api/nationforge/turn` is rate-limited per IP + session to reduce accidental abuse when exposed online. The limiter runs **after** validation so duplicate or invalid requests do not consume quota. Stuck opening beats: `stripOrphanOpeningUserMessage` removes a trailing orientation user message with no assistant prose while still `awaiting_decision`; `recoverStaleGmRunningPhase` rolls back `gm_running` if `updatedAt` is older than ~3 minutes; `onFinish` with `isAborted` pops the queued user turn and restores phase. The board no longer sets `openingBeatAutoKeySent` on benign duplicate errors (that blocked retries), uses a short cooldown instead, and treats opening 429 without throwing.

## NationForge — chronicle strip, governance, and diplomacy

- **Chronicle seat strip**: After `gameStarted`, the board shows a shared banner naming `activeNationId` (opening beat or last POV sent to the GM) plus crisis involvement names when `awaiting_decision`. Copy distinguishes single-seat vs multi-seat inflections; turn validation is still unchanged unless we add strict POV gating later.
- **`domesticScratch` (Governance and society in UI)**: Per-seat running notes on domestic policy and society. Persisted in session JSON, exposed to the GM as `governanceNotes` per nation in `buildGmSystemPrompt` (clipped with `GM_GOVERNANCE_CLIP` for prompt size). It does **not** auto-apply stats; players still ask for mechanical changes in storyline turns / tools. Still **not** included in `formatPlayerTurnMessage` (turn payload stays chronicle + crisis choices).
- **Privacy**: `filterSessionForClient` still clears `domesticScratch` for nations that are not the viewer so peers cannot read each other’s governance notes in the browser.
- **Domestic API**: `PATCH .../domestic?token=` unchanged (debounced saves, 1500 char cap, domestic rate limit).
- **`diplomaticOutreach`**: Bilateral threads (`fromNationId`, `toNationId`, `message`, optional one-shot `reply`). Stored on `GameSession`, capped (`MAX_DIPLOMACY_OUTREACH_TOTAL`). Clients only receive threads they participate in. GM sees `recentDiplomacy` (names + clipped text) in the system prompt. APIs: `POST .../diplomacy?token=` to send, `POST .../diplomacy/[outreachId]/reply?token=` for the recipient’s single reply; diplomacy rate limit separate from turns.
