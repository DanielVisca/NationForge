## Grok chat (Aetheria)

- **API key name**: Use `XAI_API_KEY` in `.env` to match [xAI](https://docs.x.ai/developers/quickstart) and Vercel AI SDK examples. The code also accepts `GROK_API_KEY` as a fallback for existing setups.

- **Responses API + history**: The UI persists full transcripts under `.data/conversations.json` for durable history and reloads. For each new *user* turn after the first reply, the server passes xAI’s `previous_response_id` (stored per conversation as `lastResponseId`) so Grok can continue the server-side chain without resending the entire transcript. Regeneration requests omit `previous_response_id` so the model sees a fresh full `input` for that interaction.

- **Vercel AI SDK**: We use `streamText` with `@ai-sdk/xai` `xai.responses(model)` so streaming, multi-step tool execution, and xAI-specific options stay aligned with current provider support.
- **Default Grok model** (`lib/xai.ts`): `grok-4-1-fast-non-reasoning` when `XAI_MODEL` is unset — fastest/cheapest tier for streamed chat and NationForge. Set `XAI_MODEL=grok-4-1-fast-reasoning` or `grok-4.20-0309-reasoning` if you want heavier reasoning at higher cost/latency.

## NationForge (game)

- **Authoritative state**: Sessions live in `.data/nationforge-sessions.json` with nations, crisis, turn log, Grok thread ids, secrets, and **`emergentEvents`** (GM-logged emergent beats, capped). The UI polls `GET /api/nationforge/sessions/[id]` for LAN-style sync (MVP).
- **Vision & emergence**: Table-facing vision is in [`README.md`](README.md) (NationForge as open-ended Aetheria sandbox). The GM prompt (`lib/nationforge/gm-prompt.ts`) encodes emergence pacing, inflection design (mix of player-driven vs emergent options), balance rules, and `declare_emergent_event` for optional logged random/semi-random shocks. `set_inflection` remains **one call** per resolution (single active crisis in state). After the opening, the prompt pushes **generative** world detail (figures, movements, institutions) with outcomes that stay **plausible** vs stats and history.
- **Stats and math**: The GM must change the six Key Stats and reserve only through the `apply_stat_deltas` tool (per-nation, L1 movement cap per invocation). Narrative stays in streamed text; the board reads numbers from the saved session after each turn.
- **Join model**: `roomCode` for discovery; `POST /api/nationforge/nations` with `{ roomCode, displayName? }` creates a nation row and returns a per-seat `token` for the **nation forge** (not an “official” table seat until `forgeComplete`). The client always opens the builder with an empty `displayName`; the server still uses a provisional `Unnamed seat {id}` until the forge **naming** step, where an AI-suggested name (overridable) is saved via `submitNationName`. No pre-forge “seat nickname” field — players jump straight into the builder. Optional `?token=` on the session URL scopes secrets to that seat. New sessions start in `lobby` with zero nations; the first crisis appears only after `gameStarted` flips when every nation in the room has completed the forge once.
- **Official seats vs builders**: `filterSessionForClient` only includes other players’ nations in `nations` when `forgeComplete`; each viewer always sees their own row even while still forging. `nationRoster` lists every slot (id, name, forge flag) so hosts can still copy tokens for in-progress builders. Session list API only joins forged names in `nationNames` and reports `nationsInForge` separately. Bilateral diplomacy requires **both** nations forged. `normalizeNation` treats **only** `forgeComplete === true` as locked-in; a missing `forgeComplete` field must **not** default to true (that hid the forge wizard and left an apparently blank table).
- **100-point nation builder**: Options and costs live in `lib/nationforge/nation-forge-catalog.ts`; spend and caps are enforced server-side in `POST .../sessions/[id]/forge`. Synergies and Key Stats mapping live in `nation-forge-resolve.ts` so the GM prompt and UI stay aligned with authoritative numbers. Mid-session joiners stay in `forgeComplete: false` until they finish the same stepped flow; `validatePlayerTurn` blocks their turns until then.
- **Review chronicle**: On the confirm step, `loadReviewNarrative` persists `reviewNarrativeMarkdown` (Grok via `generate-forge-review-narrative.ts`, Markdown) for a player-facing briefing; stats remain authoritative. Heuristic Markdown fills in when no API key. Leaving review (`back`) clears the cached narrative. The chronicle prompt must **not** narrate a multi-decade “opening era” or first-fifty-years timeline — that beat is reserved for the live GM **after** finalize when the table opens (`opening-brief-narrative.ts`); pre-lock copy stays on table-read hooks (fault lines, strengths, pressure).
- **Post-forge onboarding**: When the table first opens (`gameStarted` + a placeholder crisis, no GM prose yet), the **active seat** auto-sends one `orientationRequest` turn (`opening-brief-narrative.ts` + `validatePlayerTurn`). The GM is instructed to deliver **First 50 Years – A Brief History**, **strengths and weaknesses at Year 50**, then introduce the **first decisive event** (GM-chosen year/type), **not** resolve it, and call **set_inflection** so the real crisis prompt replaces the placeholder. Main chat and composer stay hidden until the first GM reply streams in.
- **GM “worldbuilding” UX**: While `phase === gm_running`, local `busy`, or stream text exists, the chat shows a **GM · writing** bubble at the bottom of the transcript; polls at **650ms** during `gm_running` vs **2.5s** otherwise. Benign duplicate-turn API errors (queue / still writing) trigger a refresh without scary red errors; `openingBriefInFlight` + post-success `openingBeatAutoKeySent` reduce double POSTs.
- **Persisted GM prose (reasoning models)**: The NationForge chat uses **`text` parts only** so reasoning traces are not shown to players. `gmThreadHasAssistantDelivery` / `sessionHasGmStory` treat a beat as landed when there is GM-visible prose **or** a completed GM tool row (`append_turn_log`, `apply_stat_deltas`, `no_stat_change_this_turn`, `declare_emergent_event`) so reasoning-only transcripts still unblock the opening and orphan-user cleanup. Session poll skips `setSession` when `updatedAt` is unchanged to cut UI flicker.
- **Budget dead-ends**: Every exclusive pillar includes an explicit **0 pt “underfunded / improvised”** choice so players can bank reserve or recover when earlier spends leave too little for priced tiers; those picks apply negative stat tilts in the resolver. The wizard sorts affordable options first and disables picks that exceed remaining points.
- **Back behavior**: `back` clears all selections for steps **after** the step you return to, so totals match the screen; per-step budget uses `computeSpendExcludingStep` so re-picking a pillar does not double-count. The wizard highlights the saved choice on each pillar.
- **Rate limits**: `POST /api/nationforge/turn` is rate-limited per IP + session to reduce accidental abuse when exposed online. The limiter runs **after** validation so duplicate or invalid requests do not consume quota. Stuck opening beats: `stripOrphanOpeningUserMessage` removes a trailing orientation user message with no assistant prose while still `awaiting_decision`; `recoverStaleGmRunningPhase` rolls back `gm_running` if `updatedAt` is older than ~3 minutes; `onFinish` with `isAborted` pops the queued user turn and restores phase. The board no longer sets `openingBeatAutoKeySent` on benign duplicate errors (that blocked retries), uses a short cooldown instead, and treats opening 429 without throwing.

## NationForge — board layout (2026)

- **Chat-first play surface**: After the forge, the board uses a single `max-w-3xl` flow: one **transcript** from `gmMessages` (You / GM bubbles, text parts only) plus one **Message** textarea and **Send to GM**. When `awaiting_decision` and a crisis exists, an **inflection** block renders inline in the transcript with the **full crisis prompt only** (no truncation; no lettered option list — play is open-ended prose). Optional fields (seat ribbon + nation sheet, governance scratch, public diplomacy, secret, stat asks) live under one **More with this send** `<details>` on the composer. **Session reference** is a single collapsed panel for public beat log, secrets, emergent shocks, roster, and bilateral diplomacy. Seated players with a token get **POV locked** to their nation (no POV `<select>`).
- **Markdown in chat**: `NationForgeChatMarkdown` (`react-markdown`, no raw HTML) renders CommonMark for **You**, **GM**, **inflection** crisis text, the **GM · writing** stream bubble, and the composer’s **Preview formatted** — so model-authored `**bold**`, headings, lists, and links display correctly.
- **Dictate (TTS)**: Optional checkbox above the transcript calls `POST /api/nationforge/tts` (xAI `https://api.x.ai/v1/tts`, server `XAI_API_KEY`, optional `XAI_TTS_VOICE_ID`, default `eve`) and plays returned audio in order; new GM assistant rows (Markdown stripped for speech) and new inflection prompts enqueue while playback continues. Toggle preference in `localStorage` (`nationforge-tts-enabled`); turning on primes seen message ids so the backlog is not read aloud. Per-IP rate limit on the TTS route; long copy is chunked client-side (~1.8k chars).
- **Crisis answers in prose**: For `awaiting_decision` with an active crisis (non-orientation turns), `validatePlayerTurn` requires only narrative; optional `crisisChoiceId` / `customCrisisResponse` remain for power users. `formatPlayerTurnMessage` appends a short machine line (`crisisId` + clipped prompt) when the client sends prose only so the GM always sees which inflection is active. The **You** bubble uses `playerTurnChatDisplayBody` so that suffix (and other `formatPlayerTurnMessage` appendices) never appears as player-authored text. `gm-prompt.ts` instructs the model to treat the latest user narrative as the crisis answer; `session.crisis.options` are internal to the GM, not shown in the client.

## NationForge — active seat banner, governance, and diplomacy

- **Active seat banner**: After `gameStarted`, the board shows a shared teal banner naming `activeNationId` (opening beat or last POV sent to the GM) plus crisis involvement names when `awaiting_decision`. Copy distinguishes single-seat vs multi-seat inflections.
- **`domesticScratch` (Governance and society in UI)**: Per-seat running notes on domestic policy and society. Persisted in session JSON, exposed to the GM as `governanceNotes` per nation in `buildGmSystemPrompt` (clipped with `GM_GOVERNANCE_CLIP` for prompt size). It does **not** auto-apply stats; players still ask for mechanical changes in their message to the GM / tools. Still **not** included in `formatPlayerTurnMessage` (turn payload stays the main narrative plus optional structured extras when sent).
- **Seat token vs URL (`NationForgeBoard`)**: After `Start nation forge`, `router.replace(?token=)` can update the address bar before `useSearchParams()` exposes the new query on the next render, so session `GET` ran without a token and the **Build your nation** wizard never opened until a full refresh. The board keeps a short-lived `seatTokenBridge` set synchronously when a seat token is obtained (same value as the URL) and treats `seatToken = urlToken ?? seatTokenBridge` for polling, wizard visibility, and authenticated fetches until `urlToken` catches up, then clears the bridge.
- **Seat token cache (`lib/nationforge/seat-token-cache.ts`)**: `rememberNationForgeSeat` updates `nationforge-host-tokens` (per-session nation→token map for host copy) and `nationforge-last-seat` (last seat used per session). Opening `/nationforge/[id]` without `?token=` triggers a one-time `router.replace` with the cached token when present. Session `GET` with a token but `viewerNationId === null`, or **404**, clears the last-seat entry (and prunes that nation from the host map) and strips a bad `?token=` from the URL.
- **First paint vs seat token (`NationForgeBoard`)**: The first `load()` must not run as a spectator when a token exists in the URL bar or in last-seat cache. The board resolves `?token=` from `useSearchParams` with a client fallback to `window.location.search`, hydrates `seatTokenBridge` from last-seat in `useLayoutEffect` with `flushSync` before passive effects, and only clears the bridge on real `sessionId` changes (not on initial mount) so the initial fetch and wizard gate on `seatToken` + `viewerNationId` without requiring a manual reload.

- **Privacy**: `filterSessionForClient` still clears `domesticScratch` for nations that are not the viewer so peers cannot read each other’s governance notes in the browser.
- **Domestic API**: `PATCH .../domestic?token=` unchanged (debounced saves, 1500 char cap, domestic rate limit).
- **`diplomaticOutreach`**: Bilateral threads (`fromNationId`, `toNationId`, `message`, optional one-shot `reply`). Stored on `GameSession`, capped (`MAX_DIPLOMACY_OUTREACH_TOTAL`). Clients only receive threads they participate in. GM sees `recentDiplomacy` (names + clipped text) in the system prompt. APIs: `POST .../diplomacy?token=` to send, `POST .../diplomacy/[outreachId]/reply?token=` for the recipient’s single reply; diplomacy rate limit separate from turns.
