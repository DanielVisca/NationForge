## Display preferences (global)

- **Class-based `dark`**: Tailwind `dark:` utilities use `@custom-variant dark` with the `.dark` ancestor on `<html>`, not OS media alone, so users can pick light/dark/system in-app. System mode mirrors `prefers-color-scheme` and listens for `change`.
- **Unicorn**: `data-theme="unicorn"` on `<html>` clears `.dark` (light Tailwind branch) and applies a pastel gradient on `body`; gradient motion is disabled when `prefers-reduced-motion: reduce`.
- **Text scale**: `data-text-scale` on `<html>` adjusts root `font-size` so `rem`-based UI scales without per-component edits.
- **Fonts**: Default uses Geist via `font-sans`; `readable` uses a system UI stack; `playful` uses Nunito (`--font-playful`). Settings persist in `localStorage` (`aetheria-theme`, `aetheria-text-scale`, `aetheria-font`). A small inline boot script from [`lib/display-prefs-boot.ts`](lib/display-prefs-boot.ts) applies saved values before paint to limit flash.
- **Hydration**: That boot script mutates `<html>` (`dark`, `data-theme`, etc.) before React hydrates, so `app/layout.tsx` sets `suppressHydrationWarning` on `<html>` and `<body>` (Next/React pattern for intentional pre-hydration DOM drift).

## Grok chat (Aetheria)

- **API key name**: Use `XAI_API_KEY` in `.env` to match [xAI](https://docs.x.ai/developers/quickstart) and Vercel AI SDK examples. The code also accepts `GROK_API_KEY` as a fallback for existing setups.

- **Responses API + history**: The UI persists full transcripts under `.data/conversations.json` for durable history and reloads. For each new *user* turn after the first reply, the server passes xAIÔÇÖs `previous_response_id` (stored per conversation as `lastResponseId`) so Grok can continue the server-side chain without resending the entire transcript. Regeneration requests omit `previous_response_id` so the model sees a fresh full `input` for that interaction.

- **Vercel AI SDK**: We use `streamText` with `@ai-sdk/xai` `xai.responses(model)` so streaming, multi-step tool execution, and xAI-specific options stay aligned with current provider support.
- **Default Grok model** (`lib/xai.ts`): `grok-4-1-fast-non-reasoning` when `XAI_MODEL` is unset ÔÇö fastest/cheapest tier for streamed chat and NationForge. Set `XAI_MODEL=grok-4-1-fast-reasoning` or `grok-4.20-0309-reasoning` if you want heavier reasoning at higher cost/latency.

## NationForge (game)

- **Authoritative state**: If `DATABASE_URL` is set (trimmed non-empty), sessions persist in Postgres (`nationforge_snapshot`, singleton id `sessions_v1`, JSONB payload mirroring the former file shape). Otherwise they stay in `.data/nationforge-sessions.json`. The process logs once: `[nationforge] persistence: postgres (DATABASE_URL set)` or `... local file (.data/nationforge-sessions.json)`. Neon uses `@neondatabase/serverless` `Pool` with `ws` as `neonConfig.webSocketConstructor`; locked writes run `BEGIN` Ôćĺ `SELECT ... FOR UPDATE` Ôćĺ `UPDATE` Ôćĺ `COMMIT` plus the same in-process mutation chain as the file backend. Nations, crisis, turn log, Grok thread ids, secrets, and **`emergentEvents`** (GM-logged emergent beats, capped) all live inside that snapshot. The UI polls `GET /api/nationforge/sessions/[id]` for LAN-style sync (MVP).
- **Store writes**: All mutations to `nationforge-sessions.json` run through a **global chained lock** in `lib/nationforge/store.ts` (`withLockedStore`) so concurrent finalize / turn / tool writes cannot interleave readÔÇôwrite and drop a nationÔÇÖs forge. Forge uses **`mutateSessionExclusive`** (readÔÇôapplyÔÇôwrite in one critical section). Writes use **temp file + `rename`** for atomic replace. Long GM streams still release the lock between enqueue and `onFinish`; enqueue is a second exclusive transaction so two seats cannot stampede the same row mid-open.
- **Vision & emergence**: Table-facing vision is in [`README.md`](README.md) (NationForge as open-ended Aetheria sandbox). The GM prompt (`lib/nationforge/gm-prompt.ts`) treats each next inflection as a random world event, not automatically a structured crisis: booms, crises, rebellions, discoveries, scandals, disasters, diplomacy, and quiet opportunities are all valid. `declare_emergent_event` logs random/semi-random shocks when future context matters. `set_inflection` remains **one call** per resolution (single active event in state, still named `crisis` in schema for compatibility). After the opening, the prompt pushes **generative** world detail (figures, movements, institutions) with outcomes that stay **plausible** vs stats and history.
- **Stats and math**: The GM must change the six Key Stats and reserve only through the `apply_stat_deltas` tool (per-nation, L1 movement cap per invocation). Narrative stays in streamed text; the board reads numbers from the saved session after each turn. Prompt + tool descriptions treat **pillar-aligned figurative outcomes** (e.g. ÔÇťinnovation thrivesÔÇŁ) as requiring deltas when they mean a mechanical shift; otherwise the GM must avoid implying a pillar moved and use `no_stat_change_this_turn`.
- **Future reallocation**: After play starts, players can optionally ask for stat/reserve movement in **More with this send Ôćĺ Future stat / reserve reallocation**. The request is passed to the GM as `Future stat / reserve reallocation ask:`; it is not automatic. The GM decides if the fiction supports it and, when it does, applies changes through `apply_stat_deltas` within the 10-point per-nation movement cap.
- **Join model**: `roomCode` for discovery; `POST /api/nationforge/nations` with `{ roomCode, displayName? }` creates a nation row and returns a per-seat `token` for the **nation forge** (not an ÔÇťofficialÔÇŁ table seat until `forgeComplete`). The client always opens the builder with an empty `displayName`; the server still uses a provisional `Unnamed seat {id}` until the forge **naming** step, where an AI-suggested name (overridable) is saved via `submitNationName`. No pre-forge ÔÇťseat nicknameÔÇŁ field ÔÇö players jump straight into the builder. Optional `?token=` on the session URL scopes secrets to that seat. New sessions start in `lobby` with zero nations; the first crisis appears only after `gameStarted` flips when every nation in the room has completed the forge once.
- **My Games MVP identity**: There is no account system yet. Server truth stays in `.data/nationforge-sessions.json`; the browser keeps a lightweight local profile (`nationforge-player-profile`) and enrolled seats (`nationforge-enrolled-sessions`) keyed by `sessionId`, `nationId`, and bearer `token`. The `/nationforge` landing page is `force-dynamic` and loads token-free session summaries via `listNationForgeSessionSummaries()` (same mapping as `GET /api/nationforge/sessions`, but no server-to-self HTTP ÔÇö avoids dev deadlocks). `NationForgeHomeClient` overlays local enrollments for **My games**, favorites, search, and resume links. ÔÇťForget this seat on this browserÔÇŁ deletes local enrollment/token cache only; it does not delete the server room or nation.
- **Official seats vs builders**: `filterSessionForClient` only includes other playersÔÇÖ nations in `nations` when `forgeComplete`; each viewer always sees their own row even while still forging. `nationRoster` lists every slot (id, name, forge flag) so hosts can still copy tokens for in-progress builders. Session list API only joins forged names in `nationNames` and reports `nationsInForge` separately. Bilateral diplomacy requires **both** nations forged. `normalizeNation` treats **only** `forgeComplete === true` as locked-in; a missing `forgeComplete` field must **not** default to true (that hid the forge wizard and left an apparently blank table).
- **100-point nation builder**: Options and costs live in `lib/nationforge/nation-forge-catalog.ts`; spend and caps are enforced server-side in `POST .../sessions/[id]/forge`. Synergies and Key Stats mapping live in `nation-forge-resolve.ts` so the GM prompt and UI stay aligned with authoritative numbers. Mid-session joiners stay in `forgeComplete: false` until they finish the same stepped flow; `validatePlayerTurn` blocks their turns until then.
- **Review chronicle**: On the confirm step, `loadReviewNarrative` persists `reviewNarrativeMarkdown` (Grok via `generate-forge-review-narrative.ts`, Markdown) for a player-facing briefing; stats remain authoritative. Heuristic Markdown fills in when no API key. Leaving review (`back`) clears the cached narrative. The chronicle prompt must **not** narrate a multi-decade ÔÇťopening eraÔÇŁ or first-fifty-years timeline ÔÇö that beat is reserved for the live GM **after** finalize when the table opens (`opening-brief-narrative.ts`); pre-lock copy stays on table-read hooks (fault lines, strengths, pressure).
- **Post-forge onboarding**: When the table first opens (`gameStarted` + a placeholder crisis, no GM prose yet), **each forged seat** can auto-send an `orientationRequest` for **its own nation** (`opening-brief-narrative.ts` + `validatePlayerTurn`). That wire row is **not shown in chat** (see *NationForge ÔÇö table-open brief and GM length*). The GM delivers **First 50 Years ÔÇô A Brief History**, **strengths and weaknesses at Year 50**, then the **first decisive event** (not resolved in the opening). Only the **lead/active seat** (`povNationId === activeNationId`) should call **set_inflection** to replace the placeholder; other seatsÔÇÖ parallel openings stay prose-only, backed by `set_inflection` no-ops when the placeholder is already cleared or the caller is not the active seat. Main chat and composer stay hidden until the first GM reply streams into **that** seatÔÇÖs thread.
- **Parallel GM streams**: `GameSession.gmStreamingNationIds` tracks which nations have an in-flight `streamText` turn; `validatePlayerTurn` blocks **only** the same seat from double-queueing, not the whole table. `phase` stays `gm_running` while the list is non-empty. The board fast-polls when any id is streaming and disables the composer only when **this viewerÔÇÖs** nation is in that list (plus local `busy` / stream text).
- **GM ÔÇťworldbuildingÔÇŁ UX**: While **this seat** is streaming (`gmStreamingNationIds` includes the viewer, or local `busy` / stream text), the chat shows a **GM ┬Ě writing** bubble; polls at **650ms** when **any** seat has an in-flight GM stream vs **2.5s** otherwise. Benign duplicate-turn API errors (queue / still writing) trigger a refresh without scary red errors; `openingBriefInFlight` + post-success `openingBeatAutoKeySent` reduce double POSTs.
- **Persisted GM prose (reasoning models)**: The NationForge chat uses **`text` parts only** so reasoning traces are not shown to players. `gmThreadHasAssistantDelivery` / `sessionHasGmStoryForNation` treat a beat as landed when there is GM-visible prose **or** a completed GM tool row (`append_turn_log`, `apply_stat_deltas`, `no_stat_change_this_turn`, `declare_emergent_event`) so reasoning-only transcripts still unblock the opening and orphan-user cleanup. Session poll skips `setSession` when a **play snapshot** matches (viewerÔÇÖs stat totals + reserve + a short tail of `statImpacts` ids) so we still pick up numeric changes if `updatedAt` collides or lags, without extra flicker when nothing moved.
- **Per-seat GM threads (Civ-style)**: `GameSession.gmMessagesByNationId` stores each nationÔÇÖs GM/user transcript separately; `migrateSession` copies legacy single `gmMessages` into every **forged** seat once, then clears the legacy field. `filterSessionForClient` returns only the viewerÔÇÖs thread as `gmMessages`. `buildGmSystemPrompt(session, povNationId)` adds **neighborPeers**, **recentPublicBeats**, and **tableEvents** to the JSON world snapshot (no other seatsÔÇÖ private threads). `lastGmResponseIdByNationId` continues the xAI Responses chain per seat. Forge **finalize** persists optional `forgeBriefingMarkdown` and appends a `seat_forged` row to capped `tableEvents`.
- **Budget dead-ends**: Every exclusive pillar includes an explicit **0 pt ÔÇťunderfunded / improvisedÔÇŁ** choice so players can bank reserve or recover when earlier spends leave too little for priced tiers; those picks apply negative stat tilts in the resolver. The wizard sorts affordable options first and disables picks that exceed remaining points.
- **Back behavior**: `back` clears all selections for steps **after** the step you return to, so totals match the screen; per-step budget uses `computeSpendExcludingStep` so re-picking a pillar does not double-count. The wizard highlights the saved choice on each pillar.
- **Rate limits**: `POST /api/nationforge/turn` is rate-limited per IP + session to reduce accidental abuse when exposed online. The limiter runs **after** validation so duplicate or invalid requests do not consume quota. Stuck opening beats: `stripOrphanOpeningUserMessage` removes a trailing orientation user message with no assistant prose while still `awaiting_decision`; `recoverStaleGmRunningPhase` rolls back `gm_running` if `updatedAt` is older than ~3 minutes; `onFinish` with `isAborted` pops the queued user turn and restores phase. The board no longer sets `openingBeatAutoKeySent` on benign duplicate errors (that blocked retries), uses a short cooldown instead, and treats opening 429 without throwing.

## NationForge ÔÇö table-open brief and GM length (2026)

- **Nation brief on open**: While the GM is drafting the opening beat (`gameStarted`, placeholder crisis, no assistant prose yet), the forged viewerÔÇÖs [`NationCard`](components/nationforge/NationForgeBoard.tsx) uses **`prominentForgeBriefing`** so `forgeBriefingMarkdown` is always visible (not tucked inside `<details>`). After the first GM reply (`introDelivered`), a **ÔÇťYour nation briefÔÇŁ** block above the transcript wraps the same compact card in a **collapsible `<details>`** (chevron summary + nation name) so chat stays primary; **default expanded** when no `localStorage` value exists, with preference stored per session as `nationforge-nation-brief-open:${sessionId}` (`1` / `0`).
- **Hidden auto wire message**: The first `orientationRequest` turn is still persisted as a `user` [`UIMessage`](app/api/nationforge/turn/route.ts) for the GM thread, but [`isOpeningBriefWireMessage`](lib/nationforge/opening-brief-narrative.ts) causes it to be **omitted from chat** and from [`sanitizeGmMessageForClient`](lib/nationforge/store.ts) so public session JSON never shows a fake ÔÇťplayerÔÇŁ bubble. Copy in the opening card notes that the automated request is not shown as a message from the player.
- **GM output limits**: [`POST /api/nationforge/turn`](app/api/nationforge/turn/route.ts) uses env-tunable **`NATIONFORGE_GM_MAX_OUTPUT_TOKENS`** (default **65536**, clamped 4096ÔÇô131072), higher continuation budget via **`NATIONFORGE_GM_CONTINUATION_MAX_OUTPUT_TOKENS`** (default **32768**), and **`NATIONFORGE_GM_CONTINUATION_MAX_ROUNDS`** (default **8**) so [`completeCutOffGmProseLoop`](app/api/nationforge/turn/route.ts) can stitch multiple passes after `finishReason === "length"` or incomplete tails. **`stopWhen`** uses **`stepCountIs(40)`** so tool-heavy beats are less likely to stop early. Provider APIs still enforce an absolute ceiling; override env if a model rejects a chosen value.

## NationForge ÔÇö board layout (2026)

- **Chat-first play surface**: After the forge, one **transcript** from the viewerÔÇÖs `gmMessages` slice (their seatÔÇÖs thread only; You / GM bubbles, text parts only) plus **Message** + **Send to GM**; the latest GM message is the player-facing prompt. Before intro + forged seat the shell stays **`max-w-3xl`**; after that it widens to **`max-w-6xl`** with an **`xl` two-column grid**: **Timeline** on the left; transcript, composer, and **Session reference** on the right. The chat column opens with a **nation identity block** (large name, **Government** line parsed from forge `buildNotes`, short economy / labor / military summary) and a **compact stat chip row** (reserve + six stats) instead of a separate stats sidebar. The grid is viewport-height capped with **internal scroll** in each column on large screens. Below `xl`, a single column remains; the chat card uses **max-height** instead of a tall **`min-h-[82vh]`** to cut empty page scroll. Seated players with a token get **POV locked** to their nation (no POV `<select>`).
- **Stat impact visibility**: When polling sees a **new** latest `statImpacts` id for the viewerÔÇÖs nation, a **ÔÇťStats updated this beatÔÇŁ** banner appears above the transcript (summary of deltas + reserve, dismissible, `aria-live`, auto-clears ~12s) and the **identity + stat chip** block gets a brief **ring** highlight (~2s) so `apply_stat_deltas` is obvious without rereading GM prose. When reserve or any key stat **value** changes on poll, **floating +/- labels** animate above the matching chip for ~2s while the number shows the new total.
- **Composer stats banner**: Above the **Message** textarea (sticky composer), a small **`<details>`** strip repeats reserve + full stat names with current values; expanding shows the same grid with **delta badges** for the latest GM numeric impact round (`ChatComposerStatsBanner`), complementing the identity header + animated chip row.
- **Chat cleanup and privacy boundary**: The gameplay panel now keeps player context above the transcript (ÔÇťPlaying asÔÇŁ for seated players; ÔÇťSpectating public tableÔÇŁ for unseated viewers), keeps the message composer anchored at the bottom of the chat, and shortens the main send copy so the surface feels like a chat game. Seated players do not get another nationÔÇÖs internal GM thread: `gmMessagesByNationId` never ships whole to the clientÔÇöonly the viewerÔÇÖs sanitized thread as `gmMessages`. Unseated viewers are read-only until they join with a valid seat token. Public session responses sanitize that thread, `turnLog`, assistant tool parts, and emergent event notes server-side; the browser receives public chat plus only the viewerÔÇÖs own secret/private turn-log text. Player turns are classified before persistence so clearly confidential main-message content becomes `GameSecret` state instead of raw public chat. This fixes secrecy at the data boundary, not just the UI.
- **Timeline**: The main play state includes a collapsed-by-default timeline built from public beat logs, public emergent events, and `roundIndex`. The collapsed summary still shows the active round and most recent public event, while expansion reveals older beats without becoming another prompt surface.
- **Stats and impacts**: `apply_stat_deltas` records `statImpacts` with round, nation, stat deltas, and reserve delta; aggregation for banners still **prefers the current `session.roundIndex`** when the viewer has any impacts, otherwise the highest stored round for that nation. `migrateSession` coerces missing or non-finite `roundIndex` / `reserveDelta` on legacy rows. Poll dedupe uses a **play snapshot** (viewer stat totals + tail of impact ids + **GM thread revision** `gmMessages.length:lastId`) so transcript growth invalidates skips; session `GET` uses **`fetch(..., { cache: 'no-store' })`** to avoid stale HTTP caches. The GM prompt requires **apply_stat_deltas in the same assistant turn** as prose that states numeric moves so polling players are not misled. Full forge log and stat grid remain in **Session reference** (`NationCard`).
- **Markdown in chat**: `NationForgeChatMarkdown` (`react-markdown`, no raw HTML) renders CommonMark for **You**, **GM**, the **GM ┬Ě writing** stream bubble, and the composerÔÇÖs **Preview formatted** ÔÇö so model-authored `**bold**`, headings, lists, and links display correctly.
- **Dictate (TTS)**: Optional checkbox above the transcript calls `POST /api/nationforge/tts` (xAI `https://api.x.ai/v1/tts`, server `XAI_API_KEY`, optional `XAI_TTS_VOICE_ID` server default) and plays returned audio in order; new GM assistant rows (Markdown stripped for speech) enqueue while playback continues. Allowed Grok voices: **Eve, Ara, Leo, Rex, Sal** (`voice_id` `eve` ÔÇŽ `sal`); the board stores the picker in `localStorage` (`nationforge-tts-voice`). Toggle on/off in `nationforge-tts-enabled`; turning on primes seen message ids so the backlog is not read aloud. Per-IP rate limit on the TTS route; long copy is chunked client-side (~1.8k chars).
- **Crisis answers in prose**: For `awaiting_decision` with an active crisis (non-orientation turns), `validatePlayerTurn` requires only narrative; optional `crisisChoiceId` / `customCrisisResponse` remain for power users. `formatPlayerTurnMessage` appends a short machine line (`crisisId` + clipped prompt) when the client sends prose only so the GM always sees which inflection is active. The **You** bubble uses `playerTurnChatDisplayBody` so that suffix (and other `formatPlayerTurnMessage` appendices) never appears as player-authored text. `gm-prompt.ts` instructs the model to treat the latest user narrative as the crisis answer; `session.crisis.options` are internal to the GM, not shown in the client.

## NationForge ÔÇö active seat banner, governance, and diplomacy

- **Active seat banner**: After `gameStarted`, the board shows a shared teal banner naming `activeNationId` (opening beat or last POV sent to the GM) plus crisis involvement names when `awaiting_decision`. Copy distinguishes single-seat vs multi-seat inflections.
- **`domesticScratch` (Private nation notebook)**: Per-seat running notes on domestic policy and society. Persisted in session JSON, exposed to the GM as `governanceNotes` per nation in `buildGmSystemPrompt` (clipped with `GM_GOVERNANCE_CLIP` for prompt size), and edited in **Session reference Ôćĺ Private nation notebook** rather than the turn composer. It does **not** auto-apply stats; players still ask for mechanical changes in their normal message to the GM / tools. Still **not** included in `formatPlayerTurnMessage` (turn payload stays the main narrative; legacy optional structured extras remain server-compatible but are no longer sent by the UI).
- **Seat token vs URL (`NationForgeBoard`)**: After `Start nation forge`, `router.replace(?token=)` can update the address bar before `useSearchParams()` exposes the new query on the next render, so session `GET` ran without a token and the **Build your nation** wizard never opened until a full refresh. The board keeps a short-lived `seatTokenBridge` set synchronously when a seat token is obtained (same value as the URL) and treats `seatToken = urlToken ?? seatTokenBridge` for polling, wizard visibility, and authenticated fetches until `urlToken` catches up, then clears the bridge.
- **Seat token cache (`lib/nationforge/seat-token-cache.ts`)**: `rememberNationForgeSeat` updates `nationforge-host-tokens` (per-session nationÔćĺtoken map for host copy) and `nationforge-last-seat` (last seat used per session). Opening `/nationforge/[id]` without `?token=` triggers a one-time `router.replace` with the cached token when present. Session `GET` with a token but `viewerNationId === null`, or **404**, clears the last-seat entry (and prunes that nation from the host map) and strips a bad `?token=` from the URL.
- **First paint vs seat token (`NationForgeBoard`)**: The first `load()` must not run as a spectator when a token exists in the URL bar or in last-seat cache. The board resolves `?token=` from `useSearchParams` with a client fallback to `window.location.search`, hydrates `seatTokenBridge` from last-seat in `useLayoutEffect` with `flushSync` before passive effects, and only clears the bridge on real `sessionId` changes (not on initial mount) so the initial fetch and wizard gate on `seatToken` + `viewerNationId` without requiring a manual reload.

- **Privacy**: `filterSessionForClient` still clears `domesticScratch` for nations that are not the viewer so peers cannot read each otherÔÇÖs governance notes in the browser.
- **Domestic API**: `PATCH .../domestic?token=` unchanged (debounced saves, 1500 char cap, domestic rate limit).
- **Multiplayer interaction**: Each seatÔÇÖs **private GM channel** is the primary surface for that nationÔÇÖs story; **Session reference & global affairs** surfaces public beats, shocks, roster, diplomacy, and other seatsÔÇÖ **NationCard** profiles (including optional forge briefing Markdown). Players write natural prose addressing other nations; the GM weaves world snapshot + diplomacy into **that seatÔÇÖs** thread. Bilateral threads (`DiplomaticOutreach` with full `messages` array) support private ongoing conversation between two nations. A notification badge appears on the "Structured messages" section for unread threads; a dismissible banner can announce when another seat finishes the forge. GM prompt instructs Civ-style isolation (no other seatsÔÇÖ verbatim threads), async stand-in for silent diplomacy when needed, and GM-first impact copy for the pov nation.
